# План тестирования проекта Student Progress Tracker

## 1. Цель тестирования
Проверить корректность работы основных функций приложения и выявить ошибки, а также неудобства при использовании.

## 2. Объект тестирования
Консольное Java-приложение Student Progress Tracker:
- добавление студента;
- добавление оценки;
- просмотр списка студентов и оценок;
- расчёт среднего балла;
- экспорт данных в файл.

## 3. Типы тестирования
- **Функциональное тестирование** — проверка правильной работы основных функций.
- **Тестирование удобства использования (юзабилити)** — оценка понятности меню и сообщений.
- **Smoke-тестирование** — проверка запуска и доступности основных операций.

## 4. Область тестирования
Включает:
- корректность выполнения основных сценариев;
- обработку некорректных данных (пустой ввод, неверные значения);
- понятность интерфейса.

Не включает:
- производительность;
- безопасность;
- интеграции.

## 5. Среда и инструменты
- ОС: Windows 11
- JDK: OpenJDK 17
- IDE: IntelliJ IDEA Community
- Тип тестирования: ручное (manual)
- Дополнительно: консоль, текстовый редактор для проверки экспортируемого файла.

## 6. Стратегия тестирования
1. Выполнить smoke-тест — убедиться, что приложение запускается.
2. Выполнить функциональные тесты по заранее составленным тест-кейсам.
3. Провести юзабилити-тест: дать приложению 1–3 пользователям и собрать обратную связь.
4. Зафиксировать результаты в `test_report.md` и найденные проблемы — в `issues.md`.

## 7. Критерии завершения
- Все тест-кейсы выполнены.
- Основные ошибки выявлены и задокументированы.
- Получена и кратко проанализирована обратная связь пользователей.
